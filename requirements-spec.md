# 高斯三维重建模型压缩与可视化系统
## 详细需求文档（SRS v1.0）

## 1. 文档说明

### 1.1 编写目的
本文档用于定义“基于 Flask 的高斯三维重建模型压缩与可视化系统”的详细需求，作为研发、测试、验收和后续迭代的统一基线。

### 1.2 适用范围
- 适用于当前单体 Flask Web 应用版本。
- 覆盖用户端与管理端功能、数据、流程、非功能性要求与验收标准。
- 不涉及算法模型本体训练与底层渲染引擎实现（当前为模拟流程 + 占位可视化资源）。

### 1.3 术语定义
- **压缩任务 / CompressionRun**：一次用户配置参数后触发的完整压缩执行记录。
- **日志记录**：CSV 文件中的压缩执行明细，可删除、清空、导出 Excel。
- **SuperSplat**：外部 3D 可视化编辑页面，系统支持跳转与自动导入脚本。

---

## 2. 项目背景与目标

### 2.1 背景
系统面向高斯点云 `.ply` 模型文件，提供“上传-参数配置-压缩-结果查看-日志审计-3D预览”的完整闭环。当前版本强调可演示、可管理、可回溯。

### 2.2 建设目标
1. 提供完整的用户操作主链路：注册/登录 → 上传 → 参数设置 → 压缩 → 结果查看。
2. 提供基础管理能力：管理员登录后可查看用户与最近任务，并可删除用户及关联任务。
3. 保留可审计能力：CSV 日志记录、单条删除、全部清空、Excel 导出。
4. 提供 3D 查看入口与外部工具对接能力。

### 2.3 目标用户
- **普通用户**：上传模型并获取压缩结果。
- **管理员**：管理用户与压缩任务，监控系统使用情况。

---

## 3. 系统范围

### 3.1 In Scope（本期范围）
- 用户注册、登录、退出。
- 模型上传（`.ply`）。
- 参数设置与压缩执行。
- 结果展示（指标 + 可视化资源）。
- 日志管理（查看/删除/清空/导出）。
- 3D 查看（内置 Canvas 动画 + SuperSplat 对接）。
- 管理员登录、用户管理、最近任务查看。

### 3.2 Out of Scope（本期不包含）
- 真实压缩算法后端服务编排（当前为模拟计算）。
- 多租户权限体系（当前仅用户会话 + 单管理员账号）。
- 分布式部署、灰度发布、监控告警平台。
- 国际化多语言支持。

---

## 4. 角色与权限需求

### 4.1 角色定义
1. **未登录访客**
   - 可访问：首页、用户登录、用户注册、管理员登录。
2. **登录用户**
   - 可访问：控制台、上传、参数设置、压缩结果、日志、3D 查看、退出。
3. **管理员**
   - 可访问：管理员页（用户列表、最近任务、删除用户）、管理员退出。

### 4.2 权限规则
- 用户端受 `login_required` 保护，未登录访问需重定向至用户登录页并提示。
- 管理端受 `admin_required` 保护，未登录管理员访问需重定向至管理员登录页并提示。
- 删除用户时需同步删除其关联压缩任务。

---

## 5. 业务流程需求

### 5.1 用户主流程
1. 用户注册并登录。
2. 在控制台进入“模型导入”。
3. 上传 `.ply` 文件成功后进入“参数设置”。
4. 提交参数后执行压缩。
5. 压缩完成跳转“结果页”，查看指标和可视化素材。
6. 可进一步进入“日志模块”或“3D模型查看”。

### 5.2 管理流程
1. 管理员登录后台。
2. 查看用户信息与最近 20 条压缩任务。
3. 按需删除用户（包含关联任务）。

### 5.3 日志审计流程
1. 登录用户进入日志页。
2. 查看日志表数据。
3. 可执行单条删除、全部清空、导出 Excel。

## 6. 功能需求（FR）

### FR-01 用户注册
- 输入项：用户名、邮箱、密码（必填）。
- 规则：用户名或邮箱唯一。
- 成功：创建用户记录，提示“注册成功”，跳转登录页。
- 失败：缺字段或重复时返回错误提示。

### FR-02 用户登录
- 输入项：用户名、密码。
- 成功：写入会话 `user_id`、`username`，跳转控制台。
- 失败：提示用户名或密码错误。

### FR-03 用户退出
- 清理会话并返回首页。

### FR-04 模型上传
- 支持格式：`.ply`。
- 限制：最大请求体受系统配置限制（当前 1GB）。
- 成功：保存文件至上传目录，记录 `session['uploaded_model']`，跳转参数设置。
- 失败：未选文件或格式非法时提示错误。

### FR-05 参数设置
- 参数项：
  - similarity：0.70 ~ 0.95
  - prune：0.30 ~ 0.90
  - num_codes：{64,128,256,512,1024,2048}
- 提交后保存到会话并进入压缩流程。

### FR-06 压缩执行
- 执行前置：必须已有上传模型与参数。
- 行为：模拟三阶段执行并计算指标（PSNR/SSIM/LPIPS/压缩比/耗时等）。
- 持久化：写入 `CompressionRun` 表与 CSV 日志。
- 成功：记录 `session['last_run_id']`，跳转结果页。

### FR-07 结果展示
- 展示内容：模型名、执行时间、参数值、压缩指标。
- 展示形式：表格 + 图形化摘要（条形进度）+ 本地可视化图片网格。

### FR-08 可视化接口
- 提供接口 `/api/local-visualizations`，返回模型名和可视化资源列表。

### FR-09 日志查看
- 读取 CSV 日志并渲染表格。
- 支持空数据提示（Empty State）。

### FR-10 日志清空
- 删除并重建日志文件，仅保留表头。

### FR-11 日志单条删除
- 按行号删除指定记录，不存在时提示。

### FR-12 日志导出
- 将 CSV 转换为 Excel（`report.xlsx`）并提供下载。

### FR-13 3D 查看
- 页面展示 Canvas 动画效果。
- 若存在已上传模型，展示 SuperSplat 跳转与自动导入按钮。

### FR-14 SuperSplat 导入元数据
- 接口 `/viewer/supersplat-import` 返回当前模型 URL 与编辑器 URL。

### FR-15 管理员登录
- 使用配置中的管理员账号密码校验。
- 成功写入管理员会话并进入后台页面。

### FR-16 管理员查看用户与任务
- 用户列表：按创建时间倒序。
- 任务列表：最近 20 条压缩记录。

### FR-17 管理员删除用户
- 删除指定用户及其全部关联压缩记录。
- 若删除当前登录用户，清理用户会话。

## 7. 数据需求

### 7.1 数据实体
1. **User**
   - id（PK）
   - username（唯一，必填）
   - email（唯一，必填）
   - password_hash（必填）
   - created_at

2. **CompressionRun**
   - id（PK）
   - user_id（FK -> User.id）
   - model_name
   - similarity / prune / num_codes
   - psnr / ssim / lpips
   - before_mb / after_mb / ratio / duration
   - created_at

### 7.2 文件数据
- 上传目录：`static/uploads/`
- 日志目录：`static/logs/logs.csv`
- 可视化目录：`static/visualizations/`
- 导出文件：`static/logs/report.xlsx`

### 7.3 数据完整性
- 用户名、邮箱唯一约束。
- 压缩记录必须关联有效用户。
- 日志文件缺失时自动初始化表头。

## 8. 接口与路由需求

### 8.1 页面路由
- `/` 首页
- `/register` 注册
- `/login` 登录
- `/logout` 退出
- `/dashboard` 控制台
- `/upload` 模型上传
- `/settings` 参数设置
- `/compress` 压缩执行
- `/result` 结果页
- `/logs` 日志页
- `/viewer` 3D 查看
- `/admin/login` 管理登录
- `/admin/users` 管理页
- `/admin/logout` 管理退出

### 8.2 API 路由
- `/api/local-visualizations`
- `/viewer/supersplat-import`

### 8.3 错误处理要求
- 未登录访问受限页面需重定向并提示。
- 参数错误、文件格式错误、数据缺失需友好提示。
- 外部可视化自动导入失败时需在前端提示可手动导入。

## 9. 前端与交互需求

### 9.1 视觉与布局
- 统一应用骨架（顶部栏 + 侧边栏 + 主区域）。
- 全局统一 tokens（颜色、间距、圆角、阴影）。
- 页面支持响应式布局（窄屏单栏）。

### 9.2 交互反馈
- 全局 flash 提示成功/失败/警告信息。
- 表单提交时禁用按钮并提示“处理中...”。
- 空数据页面显示 Empty State 而非空白区域。

### 9.3 可用性
- 主功能路径 ≤ 3 次点击到达关键操作页。
- 表格在小屏支持横向滚动。

---

## 10. 非功能需求（NFR）

### 10.1 性能
- 常规页面首屏响应建议 ≤ 2s（开发环境除外）。
- 压缩流程当前为模拟执行，未来真实任务需支持异步队列化。

### 10.2 安全
- 用户密码必须哈希存储。
- 会话需使用 `SECRET_KEY`。
- 上传文件名需安全化处理（防路径注入）。

### 10.3 可靠性
- 系统启动时自动初始化数据库与日志文件。
- 日志文件异常缺失时可自动恢复（重建表头）。

### 10.4 可维护性
- 文档化 UI 审计、用户流程、设计系统。
- 模板继承统一 base，减少重复代码。

---

## 11. 验收标准（UAT）

### 11.1 功能验收
1. 注册新用户后可正常登录并进入控制台。
2. 上传 `.ply` 后可进入参数设置并完成压缩。
3. 结果页能显示完整指标与可视化模块。
4. 日志支持查看、单条删除、清空、导出。
5. 管理员可登录后台、查看用户与任务、删除用户。
6. 受限页面在未授权访问时可正确重定向并提示。

### 11.2 兼容验收
- 使用 `flask run` 启动时，首次注册不应出现 `no such table: user`。
- 使用 `python app.py` 启动时功能行为一致。

### 11.3 体验验收
- 页面有统一视觉风格。
- 关键操作有即时反馈。
- 无数据状态有清晰引导。
